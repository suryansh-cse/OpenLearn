Capstone Milestone · Week 4

Build a Portfolio-Worthy CLI Tool

~3–5 hours · Advanced implementation

Milestone Capstone Goal

Build a professional CLI utility designed for your portfolio. The script must parse a dataset (CSV, JSON, or logs), model entities using Object-Oriented Programming (classes), simulate an API networking sync (e.g. querying mock HTTP statuses), write formatted analysis summaries to a file, and support CLI inputs using argument flags (such as the standard library's argparse or sys.argv).

Core Architecture Choices (Choose One)

  • Log Analyzer & Sync: Read web server logs, parse error patterns using class structures, mock an API alert request, and export a JSON error report.
  • Expense Tracker & Exporter: Input transactions via command arguments, model expenses with OOP, sync currency rates using a mock API, and output a structured CSV ledger.
  • System File Organizer: Parse folder paths, sort files by category, create folders automatically, write log logs, and print formatted directory summaries.

Technical Must-Haves

  • OOP Classes: Model data structures (e.g., Transaction, LogRecord) with attributes and methods.
  • File System Integration: Read from a file (e.g. load raw data) and write outputs (e.g. output report file).
  • Robust Error Handling: Use try-except blocks to intercept inputs that are missing or formatted incorrectly, preventing script crashes.
  • CLI Arguments: Access flags or arguments natively from the terminal using argparse.
Build & verify

Capstone Project Checklist

Develop your project in a local workspace, add a comprehensive README explaining execution commands, test extensively, and check off requirements.
