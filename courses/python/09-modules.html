Lesson 9

Modules and organizing code

~35 min

1. Object-Oriented Programming (OOP)

Object-Oriented Programming is a design paradigm used to structure software by grouping related data and behaviors into individual bundles called **objects**. In Python, you define a blueprint for these objects using the class keyword. Objects are instances of a class.

Classes contain a constructor method called __init__ which initializes the object's properties (attributes) when a new instance is created. Every instance method inside a class must accept self as its first parameter, which refers to the specific object instance calling the method.

class Student:
    # Constructor method
    def __init__(self, name, score):
        self.name = name          # Instance attribute
        self.score = score        # Instance attribute

    # Instance method
    def is_passing(self):
        return self.score >= 50

# Instantiating an object
s1 = Student("Aman", 78)
print(s1.name)             # Prints: Aman
print(s1.is_passing())     # Prints: True

2. Code Organization: Modules

As applications grow, keeping all logic in a single file becomes unmanageable. A **module** is simply a Python file (e.g., helpers.py) containing functions, classes, or variables that can be imported and reused in another script (e.g., main.py).

Python also features a vast standard library of built-in modules that you can import directly:

  • import random: Used for random numbers and shuffling sequences.
  • import math: Used for advanced mathematical operations (like square roots, trigonometric ratios).
import random
from math import sqrt

print(random.randint(1, 10))  # Prints a random integer between 1 and 10
print(sqrt(25))               # Prints: 5.0

3. Common Beginner Mistakes

⚠️ Watch out for these mistakes:

  • Forgetting 'self': Declaring a class method like def get_name():. When called on an object, Python will raise a TypeError: takes 0 positional arguments but 1 was given. Always write: def get_name(self):.
  • Forgetting parenthesis on instantiation: Writing s1 = Student instead of s1 = Student("Aman", 78). The former assigns the class blueprint itself to the variable, rather than creating a new object.
  • Circular Imports: Importing file A inside file B, and file B inside file A. This creates an import loop and throws an ImportError. Keep imports linear.
